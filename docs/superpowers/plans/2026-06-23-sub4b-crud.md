# Sub-4B — CRUD for Actions / Decisions / Blockers Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Let a user create, edit, and delete the actions, decisions, and blockers of a meeting from the detail view (today only `toggle_action` exists; decisions/blockers are read-only and have no id).

**Architecture:** Expose ids for decisions/blockers (new `Decision`/`Blocker` core structs; `MeetingDetail.decisions/blockers` become `Vec<Decision>`/`Vec<Blocker>`). Add create/update/delete to `actions_repo` and two new repos (`decisions_repo`, `blockers_repo`). Nine new Tauri commands. Frontend: inline editing in `ActionsTab` (text + owner picker + due) and inline editing of the decisions/blockers sections in `SummaryTab`. Bilingual capture = current UI language only (the other language stays NULL; AI translation is Sub-5).

**Tech Stack:** Rust + sqlx (SQLite, offline `.sqlx` cache), Tauri commands + specta bindings, React + RTK Query, vitest/RTL.

**Conventions for this plan (same as Sub-4A):**
- All NEW queries use the **unchecked** form `sqlx::query(...)` / `sqlx::query_as(...)` / `sqlx::query_scalar(...)`. Do NOT introduce new `sqlx::query!` checked macros (would require `cargo sqlx prepare`). When a task MODIFIES an existing checked query (e.g. `get_detail`), the task says so and converts it to unchecked to avoid `.sqlx` regen.
- Build/test with the LLVM/cmake env preamble (see `reference_whisper_build_toolchain`).
- Rust tests: `cargo test -p smart-noter-db` (repos) / `cargo build -p smart-noter` (commands). Frontend: `npm run test:run`, `npx tsc --noEmit`, `npm run lint`.
- IDs: actions use TEXT ids `format!("act-{}", uuid::Uuid::new_v4())` (uuid crate already a dep). Decisions/blockers use the existing `INTEGER PRIMARY KEY AUTOINCREMENT` — `create` returns the new `i64`.
- Generated `bindings.ts` and `keys.ts` are gitignored — regenerate, never commit.

```rust
/// Inserts a new action (text in the caller's UI language only; text_en stays
/// NULL until AI translation). Returns the generated id.
pub async fn create(
    pool: &SqlitePool,
    meeting_id: &str,
    text_es: &str,
    owner_participant_id: Option<&str>,
    due: Option<&str>,
) -> Result<String, DbError> {
    let id = format!("act-{}", uuid::Uuid::new_v4());
    sqlx::query(
        "INSERT INTO actions (id, meeting_id, text_es, owner_participant_id, due, done) \
         VALUES (?, ?, ?, ?, ?, 0)",
    )
    .bind(&id)
    .bind(meeting_id)
    .bind(text_es)
    .bind(owner_participant_id)
    .bind(due)
    .execute(pool)
    .await?;
    Ok(id)
}

pub async fn update(
    pool: &SqlitePool,
    action_id: &str,
    text_es: &str,
    owner_participant_id: Option<&str>,
    due: Option<&str>,
) -> Result<(), DbError> {
    sqlx::query(
        "UPDATE actions SET text_es = ?, owner_participant_id = ?, due = ? WHERE id = ?",
    )
    .bind(text_es)
    .bind(owner_participant_id)
    .bind(due)
    .bind(action_id)
    .execute(pool)
    .await?;
    Ok(())
}

pub async fn delete(pool: &SqlitePool, action_id: &str) -> Result<(), DbError> {
    sqlx::query("DELETE FROM actions WHERE id = ?")
        .bind(action_id)
        .execute(pool)
        .await?;
    Ok(())
}
```
